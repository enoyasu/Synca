# Synca 画面構成仕様書

## 1. 目的
- iPhone / iPad の可変横幅でレイアウト崩れ・見切れを防止する。
- 設定画面から日本語 / 英語を切り替え、主要UI文言に即時反映する。

## 2. 画面一覧
1. メイン画面 (`MainView`)
2. キャラクター選択シート (`CharacterSelectionView`)
3. 設定シート (`SettingsView`)

## 3. 画面遷移
1. アプリ起動 -> メイン画面表示
2. メイン画面の「CHAR/キャラ」ボタン -> キャラクター選択シート表示
3. メイン画面の「SET/設定」ボタン -> 設定シート表示
4. 各シートは「Close/閉じる」でメイン画面へ戻る

## 4. レイアウト仕様

### 4.1 共通方針
- すべての主要コンテンツは中央カラムで表示する。
- 横幅が大きい端末（iPad含む）では最大幅を制限して横に広がりすぎないようにする。
- 狭幅時は以下の順で崩れ回避する:
  1. 余白を縮小
  2. フォント縮小
  3. 行数増加（折返し）
  4. 要素を2段化

### 4.2 メイン画面 (`MainView`)
- 外側余白:
  - 幅 < 360: 10
  - 幅 < 420: 12
  - それ以外: 16
- メインカラム幅:
  - iPad等の広幅時: 最大 400
  - それ以外: 最大 440
- ヘッダー:
  - `LIVE` インジケーターは狭幅時に2段目へ退避
  - さらに狭幅時はドットのみ表示（テキスト省略）
- コントロールパネル:
  - サイドボタン幅を可変（44/48/52/56）
  - セリフバブルは狭幅ほど行数を増やして見切れ防止（最大5行）

### 4.3 感情ゲージ (`EmotionGaugeView`)
- 表示幅を受け取り、狭幅で内側余白を縮小。
- 下部の状態チップは3等分レイアウト。
- 狭幅時は短いラベル（例: `通常` / `反応強` / `特別`）に切替。

### 4.4 キャラクター選択 (`CharacterSelectionView`)
- 外側余白:
  - 幅 < 360: 10
  - 幅 < 410: 12
  - それ以外: 16
- コンテンツ最大幅:
  - 広幅時: 420
  - 通常幅: 460
- グリッド列数:
  - 利用可能幅から計算し 1〜2列で可変
- 広告削除カード:
  - 狭幅時は縦積み（情報行 + 購入ボタン）

### 4.5 設定画面 (`SettingsView`)
- 外側余白:
  - 幅 < 360: 10
  - 幅 < 410: 12
  - それ以外: 16
- コンテンツ最大幅:
  - 広幅時: 430
  - 通常幅: 470
- 言語セクションを追加し、日英切替を提供。

## 5. 多言語仕様

### 5.1 言語設定
- 保存キー: `app_language`
- 値:
  - `ja` = 日本語
  - `en` = 英語
- 変更箇所: 設定画面 > 言語セクション（Segmented Picker）

### 5.2 文言適用範囲
- メイン画面:
  - `LIVE`, `CHAR`, `SET`, `START`, `STOP`
- キャラクター選択:
  - ナビタイトル、ヘッダー、購入/キャンセル、解放確認、処理中表示、広告削除文言
- 設定画面:
  - タイトル、セクション名、スライダーラベル、情報ラベル、リセットボタン
- 感情ゲージ:
  - 状態名と状態チップ
- キャラクターカード:
  - 説明文、解放済みラベル
- セリフ:
  - `DialogueManager` で日英別データを保持し、現在言語に応じて取得

## 6. データ仕様
- `AppLanguage` 列挙体で言語を管理
- `L10n` でUI文言を一元管理
- `MainViewModel.refreshDialogue()` は現在言語で再取得

## 7. 受け入れ条件
1. iPhone (小/標準/Plus系) で横見切れがない
2. iPad (通常表示・分割表示) で主要UIが中央カラム内に収まる
3. 設定画面で言語切替後、主要UI文言が即時更新される
4. ビルドが成功する

## 8. ゲージ挙動仕様（感度連動）
- ゲージ増加は `時間あたり増加量` ベースで計算し、センサー更新周期の揺れに依存しにくくする。
- 感度最小値（0.2）では、通常利用想定で Max 到達までおおむね 30 分程度を目安とする。
- ゲージ減衰は低速に設定し、微増でも累積を感じやすくする。
- 振動イベント検出時は、ゲージ塗り部分にパルスアニメーションを表示する。
- パルスアニメーションはゲージ値そのものとは独立して、入力があったことを視覚的に示す。
